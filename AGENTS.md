# AGENTS.md - 内容自动化 Harness 核心大脑

> 本文件定义了 Agent 产出内容的所有规则和质量标准

---

## 账号定位

**主题**：AI 产品体验设计  
**目标读者**：25-35 岁的产品设计师、体验设计师、AI 产品从业者  
**账号调性**：犀利、有洞察、反直觉

---

## 硬性规则（质量门禁）

### 1. 开头规则
- 第一句必须是**结论**或**反直觉观点**
- 禁止铺垫、禁止背景介绍、禁止"近年来"

### 2. 内容规则
- 每篇必须包含至少 **1 个真实案例** 或 **具体数据**
- 案例必须带来源（公司/产品/时间）
- 数据必须带出处

### 3. 禁用词汇
以下词汇严禁出现：
- "值得注意的是"
- "总的来说"
- "毋庸置疑"
- "众所周知"
- "不难发现"

### 4. 数据来源铁律（零幻觉政策）
**所有案例、数据、引用必须真实可查，禁止编造。**

**硬性要求：**
- 每个案例必须附带真实来源（公司名称、时间、具体事件）
- 每个数据必须附带出处链接或报告名称
- 不确定的信息 → 直接删除，不要写
- 无法验证的引用 → 换案例或标注"待核实"

**来源优先级：**
1. 官方发布（公司博客、财报、新闻发布会）
2. 权威媒体（TechCrunch、The Verge、36氪等）
3. 行业报告（Gartner、IDC、Stack Overflow Survey 等）
4. 学术研究（ArXiv、IEEE、ACM 等）

**禁止行为：**
- ❌ 编造调研数据（如"78% 的用户认为..."）
- ❌ 虚构案例细节（如"某头部电商公司..."）
- ❌ 模糊时间表述掩盖过时数据（如用"近年"代替具体年份）
- ❌ 推断性表述伪装成事实（如"研究表明..."无出处）

**执行标准：**
如果无法提供可点击验证的来源链接，该内容不得出现。

### 4. 字数规范
- **公众号文章**：800-1500 字
- **X Thread**：10-15 条推文

---

## 内容结构模板

### 标准结构（三段式）

**开头（10%）**：1 句话抛结论  
→ 反直觉优先，制造认知冲突  
→ 示例："90% 的 AI 产品都在犯同一个错误：功能越多越好。"

**中段（70%）**：案例/数据支撑  
→ 2-3 个案例或数据点  
→ 每个案例包含：背景 → 冲突 → 结果

**结尾（20%）**：可执行的行动建议  
→ 1 个具体、可立即执行的动作  
→ 避免泛泛而谈的"要关注"

## 平台适配规则

### 公众号
- 标题：15-25 字，带数字或对比
- 结构：三段式，案例为主
- 风格：深度、有洞察

### X (Twitter)
- Thread 结构：钩子 → 论点 → 案例 → 结论
- 每条推文：1 个观点，独立成文
- 风格：简洁、犀利、可传播

---

## 迭代规则（基于数据反馈）

每周日审查数据表，自动调整：
1. **高互动选题** → 加大同类选题比例
2. **低互动选题** → 减少或改变切入角度
3. **新出现热点** → 纳入选题池

数据字段：
- 标题
- 选题类型（案例分析/方法论/热点解读）
- 发布日期
- 阅读量
- 互动率（点赞+评论+转发）
- 表现评级：🔥 跑得好 / ➖ 跑得一般 / ❌ 跑得差
