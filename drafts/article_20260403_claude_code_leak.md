# Claude Code 源码泄露：Anthropic 的工程边界，才是创业者的地图

57MB 的映射文件、51.2 万行 TypeScript 代码、1906 个未混淆源文件——Anthropic 四年磨一剑的 Claude Code 家底，因为一个工程师的配置疏漏，一夜之间成了公共财产。

这不是灾难，是一份**创业工具书**。

---

## 被低估的 Harness 工程深度

外界长期有个误解：Claude Code 不过是"模型+提示词+工具调用"的轻量包装，创业公司几个月就能搭出 MVP。

泄露的代码给出了清醒的答案。

Claude Code 不是 Claude 的 CLI 工具，而是 **Anthropic 的 Agent 操作系统底座**。query.ts 核心查询循环，800 多行的状态机，处理 7 种消息类型。三个工程细节说明了真实重量：

**四层上下文压缩**——不是对话太长就粗暴截断，而是递进机制，优先保持粒度。这说明 Anthropic 押注的核心场景是：Agent 连续工作数小时甚至数天，上下文管理是命门。

**流式工具并行执行**——模型生成和工具调用真正并行，而不是排队。优化的是 Agent 同时做很多事的吞吐量。

**多模式运行与递归计费**——交互式、嵌入式、持久后台三种模式，计费精细到子调用级别。这是平台级的复杂度。

生产级的 Harness 是系统工程，不是一两个工具的组合。

---

## 两个战略信号：KAIROS 与 PROACTIVE

泄露代码中散落着 17 个 Feature Flag，其中两个方向不容忽视。

**KAIROS**——持久后台代理，藏在隐藏模式里。Agent 的终局形态不是人坐在电脑前跟它对话，而是它在后台自主运行。状态持久化、任务恢复、长时间运行的资源管理，全都是为"无人值守"准备的。

**PROACTIVE**——主动式 AI。传统模式是人下指令、AI 执行。PROACTIVE 要反过来：AI 主动发起行动，人负责审批。交互范式的根本翻转。

KAIROS 定义了 Agent"在哪里跑"，PROACTIVE 定义了 Agent"谁说了算"。

叠加起来是一幅清晰图景：Agent 不等你说话，自己决定干什么，在后台持续执行，你只需要偶尔看一眼、批准或否决。

对创业者来说，这是时间窗口的信号。Anthropic 大体要沿着这个方向进化，但现在没有成型产品。在 Anthropic 发大招之前，在垂直领域建立先发优势的窗口还开着。

## 边界之外的三片荒野

这份泄露代码最大的价值，是让你看清 Anthropic 的疆域——哪些是筑好的内核，哪些是尚未涉足的荒野。

**已筑好的内核，不要碰**：
- 通用编排（query.ts 的 800 行状态机）
- 基础工具接入（MCP 协议标准化）
- 四层上下文压缩（工程护城河）
- 递归成本追踪（平台级复杂度）

在这些领域创业，等于正面撞 Anthropic 的操作系统底座。

**但有三片荒野，是机会**：

**1. 训练环境**
泄露代码中 taskBudget 与 maxTurns 只是雏形，没有完善的沙箱，也没有专业的 rollout 基础设施。Agentic RL 的最大瓶颈不是算法，是环境质量。专业 RL 沙箱、代码执行环境、浏览器模拟器——供需错配最严重的窗口，就是现在。

**2. 垂直行业技能**
Claude Code 提供了通用技能加载框架，但框架和行业知识是两层工程。法律文书的引用格式、医疗记录的合规要求、金融模型的风控逻辑——这些东西不是通用框架能解决的。Anthropic 的工程师懂状态机，但不懂医疗合规流程。

**3. 独立评估**
代码有 logEvent，Agent 每一步操作都会被记录。但银行不会信任被审计方自己的工具做合规检查。第三方行为审计、跨模型成本归因、合规检查引擎——Agent 越自主，缰绳（Harness）的价格越贵，但缰绳不能由被驾驭者自己打造。

## 一个判断标准

下次你看到"AI+垂直行业"的创业机会，先问：

**这件事 Anthropic 会不会自己做？**

如果它需要深厚的行业知识、复杂的合规流程、或者与模型层无关的基础设施——Anthropic 大概率不会做。这是创业者的空间。

如果它是通用编排、上下文管理、工具调用标准化——Anthropic 已经做了，而且做得极深。别碰。

Claude Code 的泄露，让外界第一次看到 Agent 操作系统底座的实际边界。

**看到边界，和在边界之外建出东西，是两件完全不同的事。**

但至少有地图了。
