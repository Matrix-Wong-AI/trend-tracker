🚀 Content Automation Harness

AI 驱动的内容自动化工作流 - 从选题到发布的完整闭环

1
📥 选题输入 (Input Gate) 已配置
用户提供真实链接或话题，确保信息源可追溯。零幻觉政策：所有内容必须基于可验证来源。
  • 用户每日扫描信息源（即刻/推特/Newsletter）
  • 提供可点击验证的链接
  • 我尝试抓取，失败则用户粘贴核心内容
  • 禁止：编造数据、模糊来源、过时信息
2
🧠 系统规则 (AGENTS.md) 已配置
核心大脑文件，定义账号定位、硬性规则、内容结构。每次写作时作为 System Prompt 注入。
  • 账号定位：AI 产品体验设计 / 25-35岁设计师
  • 开头规则：第一句必须是结论或反直觉观点
  • 内容规则：每个案例必须带真实来源
  • 禁用词汇："值得注意的是"、"总的来说"等
  • 字数规范：公众号 800-1500 字
  • 数据来源铁律：零幻觉政策，全部可验证
3
✍️ 内容生成 (Generation Engine) 已验证
基于 AGENTS.md 约束，将选题转化为符合规范的文章。三段式结构：开头抛结论 → 中段案例支撑 → 结尾行动建议。
  • 输入：选题 + AGENTS.md System Prompt
  • 生成：按三段式结构产出草稿
  • 自检：硬规则校验（开头/禁用词/字数）
  • 输出：符合规范的 Markdown 文件
  • 保存：content-harness/drafts/
4
👤 人工审核 (Publish Gate) 待配置
人工终审节点。你检查文章质量，确认或要求修改。这是质量的最后把关。
  • 你审核草稿（内容准确性/风格匹配）
  • 选择：直接发布 / 修改后发布 / 放弃
  • 修改意见补充进 AGENTS.md 迭代规则
  • 需要配置：公众号 Token（自动上传草稿箱）
5
📊 数据追踪 (Feedback Loop) 已配置
每篇文章记录数据，形成内容数据库。每周分析，驱动规则迭代。
  • 记录字段：标题、选题类型、发布日期
  • 数据字段：阅读量、互动率、表现评级
  • 存储：content_tracking.csv（可迁飞书多维表）
  • 每周分析：高互动选题 → 加大比例
6
🔄 规则迭代 (AGENTS.md 更新) 持续进行
基于数据反馈，持续优化 AGENTS.md。用数据替代感觉，让 Harness 自动进化。
  • 高互动选题 → 调整选题权重
  • 修改原因 → 补充进硬性规则
  • 风格反馈 → 更新账号调性描述
  • 每周末：数据复盘 → 规则迭代
